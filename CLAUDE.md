## Project Overview
Wafer surface anomaly detection system for semiconductor manufacturing ().
Analyses ZA/ZE side surface topography from Zygo `.wrk` files across process steps
(before IBF → after IBF → after coating → after HI bonding → after Z polishing).

**Core goal:** Given a wafer surface measurement, detect and quantify localised
anomalies — regions where the surface deviates from what the process should produce.

## The Anomaly Detection Problem — Core Physics

Every wafer diff map has two components:

```
diff_map = process_component + anomaly_component

process_component:  smooth, low spatial frequency, varies slowly across the wafer
                    → well described by a low-degree 2D polynomial
                    → this is EXPECTED — every wafer has it

anomaly_component:  localised spatial deviation, does NOT fit the smooth background
                    → this is what we want to detect and quantify
```

**Key insight:** Even wafers labelled "Normal" have spatial variation — it is
process-induced and expected. The anomaly is defined as deviation from the smooth
process background, NOT deviation from zero or from an average wafer.

This means:
- Comparing to an "average normal wafer" is wrong — every wafer is slightly different
- A pixel-level autoencoder trained on normal images is unreliable — it partially
  learns the anomalous patterns too if they appear consistently in similar positions
- The correct approach is per-image decomposition: remove the smooth process
  component, examine what remains

## Baseline Estimation Methods (analysis/prediction.py)

### Method 1: Robust Polynomial (RECOMMENDED — default in Tab 2)
**`predict_normal_robust_polynomial(z, degree=5, n_iter=5, k_sigma=2.0)`**

IRLS (Iteratively Reweighted Least Squares) with Huber weights:
1. Fit polynomial to all valid pixels (unweighted)
2. Compute residuals → robust sigma via MAD × 1.4826
3. Huber weights: `w_i = min(1, k_sigma × sigma / |r_i|)` — defect pixels get w < 1
4. Weighted least-squares refit
5. Repeat 3–4 for n_iter iterations → defect pixels converge to near-zero weight
6. Final polynomial = process component only
7. Anomaly mask = pixels where `|residual| > k_sigma × sigma_final`

**Why best:** No region selection needed. Physically correct. Works on a single image.
`k_sigma=2.0` catches subtle anomalies; increase to 3.0 for less sensitive detection.
Runtime < 1 second. No training data required.

### Method 2: Polynomial (manual region)
**`predict_normal_polynomial(z, anom_col_start, anom_col_end, degree=5)`**

Fits polynomial to the user-selected CLEAN region only (box-draw on the Original
image), then extrapolates into the anomalous region. Requires manual region selection.
Good for validation/comparison with Robust Poly.

### Method 3: Gaussian low-pass
**`predict_normal_gaussian(topo, fwhm_m, anom_col_start, anom_col_end)`**

Applies `topo.gauss_low_pass(fwhm_m=fwhm_m)` globally. Removes high-frequency
content but misses large-scale drift. Influenced by the anomaly itself (global filter).
Provided as comparison only.

### Method 4: Rectangular Inpainting (Tab 2 — Anomaly Removal)
**`inpaint_region(z, row_start, row_end, col_start, col_end, degree=5)`**

Fits polynomial to all valid pixels OUTSIDE a user-selected rectangle, evaluates
on full grid, replaces rectangle pixels with prediction. Returns `(cleaned_z, predicted_z)`.
Used in Tab 2 for interactive anomaly removal via box-draw.

### Shared utilities
- `compute_metrics(anomaly_z, anom_mask)` → RMS, PV, Max |dev|, Mean, anomaly area %
- `_poly_design_matrix(rows, cols, n_rows, n_cols, degree)` → 2D polynomial features,
  coordinates normalised to [−1, 1] for numerical stability
- `_weighted_lstsq(A, z, weights)` → weighted least squares via sqrt-scaling

## ML Roadmap — Specific Plan

### Why a simple autoencoder is NOT the right approach here
Training an autoencoder on normal images and using reconstruction error as anomaly
score fails because:
1. The process signature is spatially consistent across wafers — the autoencoder
   learns to reconstruct even subtle defects that appear in similar locations
2. "Normal" is not well-defined — every wafer has its own process variation
3. Reconstruction error conflates process variation with genuine defects

### The correct ML pipeline (per-image physics first, ML on top)

```
Step 1 — Physics (per image, no labels needed):
    Robust polynomial fit (IRLS) → anomaly_z = residual map
    This removes the process component cleanly.

Step 2 — Feature extraction from residual map (no labels needed):
    - Spatial extent of high-residual region (mm²)
    - Peak residual (nm)
    - Shape descriptor: aspect ratio, elongation, edge-alignment
    - Location: distance from ZA/ZE boundary, distance from edge
    - RMS of residual in top-k% pixels
    - Spatial frequency content (FFT peak)

Step 3 — Classifier on features (needs ~50-100 labeled examples):
    Input: feature vector from Step 2
    Output: Normal / Anomaly + confidence score
    Model: logistic regression or random forest (NOT deep learning)
    Fully explainable: "flagged because peak residual = 38 nm at ZE boundary"

Step 4 (future, Phase D — needs 200+ labeled images):
    U-Net trained on RESIDUAL maps (not raw images)
    Input: anomaly_z from Step 1
    Output: anomaly probability map per pixel
```

### Data for training (generated_files.parquet)
- 535 ZA/ZE diff maps: `after HI bonding − after coating` (cleanest signal)
- ~80-90% expected normal → ~400-480 clean training examples
- Augmentation: horizontal flip, small rotation (±5°), Gaussian noise
- For Steps 1-2: NO labels needed → can start immediately
- For Step 3: need ~50 confirmed Normal labels from Tab 1 labeling
- Tab 1 labels feed directly into Step 3 training set

### Why 535 images is enough
- Step 1 (IRLS fit): 1 image, instant
- Step 2 (feature extraction): 1 image, instant
- Step 3 (logistic regression / random forest): 50-100 labeled examples sufficient
- Step 4 (U-Net on residuals): 200+ labeled residual maps needed — future

## Dash App — Tab 2: Anomaly Removal (restructured 2026-03-06)

### Purpose
Interactive anomaly removal from diff maps. User selects a diff map from a table,
draws a box around the anomaly on the Original image, and the Cleaned image auto-updates
with the anomaly region replaced by a polynomial surface fit from surrounding pixels.

### Data source
Same as Tab 3: `diff_map_files_HI` — 613 ZA/ZE diff maps from `generated_files.parquet`.
Loaded via `fl.read_zygo(path)` → Topography object (fastlibrary).

### Layout (side-by-side, same pattern as Tab 3)
- **Left panel (38%):** Side filter dropdown + DataTable (Side, Date, File)
- **Right panel (62%):**
  1. File title
  2. Controls: Poly degree dropdown + Gauss FWHM slider
  3. **Image 1 — Original Diff Map** (Gaussian smoothed, box-select enabled for metrics)
  4. Info line (IRLS summary for whole image)
  5. **Image 2 — Cleaned** (IRLS Robust Poly — anomaly removed from whole image)
  6. **Region metrics panel** — appears when user draws a box on Image 1
  7. **Metric legend** — collapsible "What do these metrics mean?" with descriptions

### How it works
1. File loaded via `fl.read_zygo(path)` → Topography object
2. Gaussian smoothing: `topo.gauss_low_pass(fwhm_m=...)` applied (controlled by FWHM slider)
3. Image 1 shows Gaussian-smoothed diff map
4. Image 2 auto-computes IRLS robust polynomial on the smoothed data → shows the
   process baseline with anomaly removed (whole image, no selection needed)
5. User draws a box on Image 1 → region metrics panel compares that box area
   between Image 1 (smoothed original) and Image 2 (baseline), computing deviation

### Region metrics (box-select on Image 1)
Deviation = smoothed original − IRLS baseline, computed only in the selected box.
- **PV (Peak-to-Valley):** total height range of anomaly (nm)
- **RMS:** root-mean-square deviation — standard semiconductor surface quality metric (nm)
- **Mean deviation:** signed average — bump (+) or dip (−) (nm)
- **Max |deviation|:** worst-case single point (nm) — used for pass/fail specs
- **Std deviation:** spread of deviation values (nm)
- **Anomaly area:** physical area where |deviation| > 10 nm (mm² and % of box)
- **Anomaly volume:** sum of |deviation| × pixel area (nm·mm²) — total displaced material
- **Max gradient:** sharpest edge (nm/pixel) — sharp = mechanical defect, gradual = process drift
- Physical dimensions computed from 442×28 mm² wafer scan area

### Controls
- **Poly degree** dropdown (1-6, default 5) — polynomial degree for IRLS baseline
- **Gauss FWHM slider** (1-20 mm, default 5) — Gaussian low-pass filter strength

### Key callbacks
- `filter_t2_table()` — side filter for table
- `load_t2_original()` — row click or FWHM change → `fl.read_zygo` + Gaussian → Image 1
- `update_t2_cleaned()` — path or controls change → Gaussian + IRLS robust poly → Image 2
- `compute_t2_region_metrics()` — box drawn on Image 1 → compares region vs baseline → metrics

### Stores
- `dcc.Store(id="t2-full-path")` — stores the full file path of the selected diff map
