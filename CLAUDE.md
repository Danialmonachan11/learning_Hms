## Project Overview
Deep learning-based wafer defect prediction system for semiconductor manufacturing (ASML).
Uses a U-Net architecture to predict surface topography from masked wafer images.

- **Training:** 256×512 resolution, L1 + SSIM loss, batch 8, lr 1e-4

## Dash App — Tab 2: Anomaly Analysis (DONE — 2026-03-04)

### What it does
For any selected measurement, fits a 2D polynomial to the clean (non-anomalous)
region and extrapolates to predict what the surface looks like without defects.
Shows three stacked full-width images: Original | Predicted Normal | Anomaly Map,
plus quantitative metrics.

### Why 2D polynomial (not mirror symmetry / Gaussian / GP)
- **Wafer surfaces are smooth optical surfaces**: tilt + curvature + higher-order terms
  are naturally described by low-degree polynomials.
- **Mirror symmetry** assumes left↔right symmetry — not true for real polished surfaces.
- **Gaussian low-pass** removes only high-frequency content, misses large-scale drift.
  Offered as a comparison method (Method selector in UI).
- **Gaussian process regression** too slow for ~77k pixels without sparse approximation.
- **scipy.linalg.lstsq** solves the polynomial fit in < 1 second; no new dependency
  (scipy is already a transitive dep via torch/sklearn).

### Anomalous region
- User-defined via a **range slider** (0–100% of image columns).
- Default: ZA → [60%, 100%], ZE → [0%, 40%] (auto-set when Side changes).
- Polynomial is fit to all pixels OUTSIDE the selected range (clean region).
- Metrics are computed on the selected range (anomalous region) only.

### Computation model
- No pre-trained model file; calculated fresh per Analyse click.
- `fl.read_zygo(source_path)` → `topo.z_map * 1e9` (nm)
- Design matrix: all monomials x^i * y^j where i+j ≤ degree, coords normalised to [-1,1]
- `scipy.linalg.lstsq` → polynomial coefficients → evaluate on full grid → predicted_z
- `anomaly_z = actual_z − predicted_z`

### Metrics (computed on anomalous-region pixels only)
- **RMS**: root-mean-square deviation (nm)
- **PV**: peak-to-valley range (nm)
- **Max |dev|**: maximum absolute deviation (nm)
- **Mean**: mean deviation (nm, signed — positive = surface higher than predicted)
- **Anomaly area %**: fraction of anomalous-region pixels with |deviation| > 10 nm

### Colorscales
- Original + Predicted: DARK_RAINBOW (same scale, comparable)
- Anomaly map: `RdBu_r` diverging (blue = below predicted, red = above)

### Future: replace polynomial with PCA statistical model
Once Tab 1 labeling produces enough Normal images, replace polynomial extrapolation
with a PCA-based statistical model trained on the labeled normal set. The API in
`analysis/prediction.py` is designed to be drop-in replaceable.
